# 7. Обоснование выбора стека технологий

## Frontend

### Язык: Java

**Причины выбора:**
- Зрелый и стабильный язык
- Широкая экосистема
- Поддержка Android SDK
- Большое сообщество

### Фреймворк: Android SDK + Material Components

**Причины выбора:**
- Нативная разработка
- Высокая производительность
- Соответствие Material Design
- Лучшее взаимодействие с системой

---

## Backend и база данных

### Firebase (Backend-as-a-Service)

#### Firebase Authentication
- Готовые механизмы регистрации и входа
- Безопасное хранение учетных данных
- Поддержка email/password

#### Cloud Firestore
- NoSQL база данных
- Real-time обновления
- Гибкая структура данных
- Высокая масштабируемость

#### Firebase Storage
- Хранение изображений блюд
- Хранение пользовательских аватаров

**Причины выбора Firebase:**
- Быстрая разработка
- Масштабируемость
- Бесплатный тариф для MVP
- Минимальная серверная логика

---

## Внешние API

### Spoonacular API

**Назначение:**
- Генерация рецептов
- Получение данных о питательности
- КБЖУ и ингредиенты

**Причины выбора:**
- Большая база рецептов
- Поддержка нутриентов
- Удобный REST API

**Ограничения:**
- 150 запросов/день (бесплатный тариф)
