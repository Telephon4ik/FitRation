# 7. Обоснование выбора стека технологий

### Язык: Java

**Причины выбора:**
- Широкая экосистема
- Поддержка Android SDK

### Фреймворк: Android SDK + Material Components

**Причины выбора:**
- Нативная разработка
- Высокая производительность
- Соответствие Material Design
- Лучшее взаимодействие с системой
- 
### Firebase 

#### Firebase Authentication
- Готовые механизмы регистрации и входа
- Безопасное хранение учетных данных
- Поддержка email/password

#### Cloud Firestore
- NoSQL база данных
- Real-time обновления
- Гибкая структура данных
- Высокая масштабируемость

**Причины выбора Firebase:**
- Быстрая разработка
- Масштабируемость
- Бесплатный тариф
- Минимальная серверная логика

---

## Внешние API

### Spoonacular API

**Назначение:**
- Генерация рецептов
- Получение данных о питательности
- КБЖУ и ингредиенты

**Причины выбора:**
- Большая база рецептов
- Поддержка нутриентов

**Ограничения:**
- 150 запросов/день (бесплатный тариф)

## Библиотеки

- **Glide** — загрузка и кэширование изображений
- **Gson** — парсинг JSON
- **OkHttp** — HTTP-клиент для API запросов
- **Material Components** — современный UI

## Система контроля версий

- **Git** — контроль версий
- **GitHub** — удалённый репозиторий

**Причины выбора:**
- Стандарт индустрии
- Удобная совместная работа
