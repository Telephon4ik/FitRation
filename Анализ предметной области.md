# 1. Анализ предметной области

**Предметная область:**  
Система управления питанием

## Ключевые процессы
- Регистрация и авторизация пользователей с ролями (USER/COACH)
- Управление профилем
- Планирование питания:
    - Автоматическая генерация рациона
    - Отметка о потреблении пищи
- Взаимодействие тренер-клиент:
    - Система заявок на привязку
    - Просмотр рациона клиентов

## Сущности
- **User** — пользователь
- **Meal** — приём пищи
- **CoachRequest** — заявка на связь тренер-клиент
- **Nutrition** — объект с данными о питательности

## Требования
- Поддержка двух ролей: обычный пользователь и тренер
- Обновление данных через Firebase
- Раздельный интерфейс для разных ролей
