# 1. Анализ предметной области

**Предметная область:**  
Система управления питанием и диетологическим коучингом

## Ключевые процессы
- Регистрация и аутентификация пользователей с ролями (USER/COACH)
- Управление профилем (антропометрические данные, цели, нормы КБЖУ)
- Планирование питания:
    - Автоматическая генерация рациона через Spoonacular API
    - Ручное добавление/редактирование блюд
    - Отметка о потреблении пищи
- Взаимодействие тренер-клиент:
    - Система заявок на привязку
    - Просмотр рациона клиентов
    - Комментирование блюд
    - Просмотр статистики
- Отслеживание прогресса:
    - Мониторинг потребления КБЖУ
    - Сравнение с целями

## Сущности
- **User** — пользователь (uid, email, name, role, publicId, coachId, антропометрия)
- **Meal** — приём пищи (дата, тип, название, КБЖУ, изображение)
- **CoachRequest** — заявка на связь тренер-клиент
- **Nutrition** — объект с данными о питательности

## Требования
- Поддержка двух ролей: обычный пользователь и тренер
- Интеграция с внешним API для генерации рецептов
- Real-time обновление данных через Firebase
- Раздельный интерфейс для разных ролей
