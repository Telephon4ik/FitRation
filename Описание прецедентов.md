# 4. Описание прецедентов

## 1: Регистрация нового пользователя
**Актор:** Новый пользователь  
**Цель:** Создать учетную запись

**Шаги:**
1. Открыть приложение
2. Нажать "Зарегистрироваться"
3. Заполнить форму (имя, email, пароль, роль)
4. Для USER — заполнить профиль (вес, рост, цели)
5. Подтвердить регистрацию

**Исключения:**
- Email уже существует
- Пароль менее 5 символов
- Нет интернет-соединения

---

## 2: Генерация дневного рациона
**Актор:** Пользователь (USER)  
**Цель:** Получить план питания на день

**Шаги:**
1. Открыть вкладку "Рацион"
2. Нажать "Сгенерировать рацион"
3. Дождаться обработки
4. Просмотреть предложенные блюда

**Исключения:**
- Ошибка API Spoonacular
- Нет интернет-соединения

---

## 3: Привязка к тренеру
**Актор:** Пользователь (USER)  
**Цель:** Найти и привязаться к тренеру

**Шаги:**
1. Открыть профиль
2. Нажать "Отправить заявку тренеру"
3. Ввести ID тренера (формат FR-XXXXXX)
4. Отправить заявку
5. Дождаться подтверждения

**Исключения:**
- Неверный ID тренера
- Уже есть активная заявка
- Тренер отклонил заявку

---

## 4: Просмотр клиентов
**Актор:** Тренер (COACH)  
**Цель:** Увидеть список своих клиентов

**Шаги:**
1. Открыть вкладку "Клиенты"
2. Просмотреть список
3. Выбрать клиента для деталей

**Исключения:**
- Нет клиентов
- Ошибка загрузки данных

---

## 5: Комментирование блюда клиента
**Актор:** Тренер (COACH)  
**Цель:** Дать рекомендацию по питанию

**Шаги:**
1. Открыть рацион клиента
2. Выбрать блюдо
3. Нажать "Добавить комментарий"
4. Ввести рекомендацию
5. Сохранить

**Исключения:**
- Нет прав доступа
- Ошибка сохранения
